# 问题设定

## 1. 背景

在三维集成电路（3D-IC）中，多层硅片垂直堆叠，通过**硅通孔（TSV, Through-Silicon Via）**实现互连。TSV是穿过硅衬底的垂直电气连接，能够在层间提供高带宽、低延迟的通信。

然而，TSV容易受到制造缺陷和运行时退化（如电迁移、热应力、空洞形成）的影响。TSV失效会中断层间信号传输路径。为确保可靠性，需要部署**冗余TSV**作为备用单元，在失效时替代故障TSV。

TSVRA（**TSV冗余架构**，TSV Redundancy Architecture）是一个模拟系统，用于在各种失效场景下评估冗余策略和路由算法的性能。

## 2. 物理模型

### 2.1 3D-IC结构

芯片被建模为按层组织的**三维TSV节点网格**：

- **网格尺寸**：N × N × L
  - N = 4 × `grid_factor`（默认：grid_factor = 4，即 N = 16）
  - L = `num_layers`（默认：2）
- **坐标系统**：(x, y, z)，其中 x, y ∈ [0, N-1]，z ∈ [0, L-1]
- **TSV总数**：N² × L（默认
：512）

网格中每个节点代表位于 (x, y, z) 位置的一个**逐层TSV单元**。TSV是相邻两层之间的垂直连接，**并非**贯穿整个芯片从顶到底的通道。(x, y, z) 处的TSV失效仅影响该特定层的连接；同一 (x, y) 位置其他层的TSV不受影响。

### 2.2 TSV类型

系统中有两种TSV：

| 类型 | 功能 | 布局位置 |
|------|------|----------|
| **普通TSV** | 承载层间常规信号传输 | 每个4×4区域中非顶角位置 |
| **冗余TSV** | 备用单元，仅在绕过失效TSV时激活 | 每个4×4区域的四个顶角位置 |

冗余TSV**不承载**常规流量，专门作为备用保留。

### 2.3 冗余分组（4×4区域）

网格被划分为不重叠的**4×4区域**（每层包含16个TSV位置）：

- 区域 (i, j) 覆盖 x ∈ [4i, 4i+3]，y ∈ [4j, 4j+3]
- 对于16×16网格：4×4 = 16个区域/层
- 每个区域在四个顶角有**4个冗余TSV**：(4i, 4j)、(4i+3, 4j)、(4i, 4j+3)、(4i+3, 4j+3)

顶角TSV在相邻区域之间**共享**（例如位置 (3, 0) 同时是区域 (0,0) 和相邻区域的顶角），实现跨区域冗余覆盖。

> **注意**：当前基于4×4顶角的冗余方案是一种特定策略。模拟器设计目标是在未来开发中评估和比较不同的冗余布局策略。

### 2.4 信号流向

所有信号请求从源位置（任意层）**向上路由到顶层**（z = L-1）。这模拟了I/O焊盘或键合接口位于顶层芯片的物理场景，底层信号必须到达顶层。

## 3. 移动与延迟模型

### 3.1 移动规则

从任何TSV节点，信号可以向**6个方向**移动：

- **4个水平移动**（同层内）：±x、±y
- **2个垂直移动**（层间）：±z

### 3.2 延迟模型

两种移动类型具有本质不同的物理特性：

| 移动方式 | 延迟 | 物理含义 |
|----------|------|----------|
| **垂直**（通过TSV换层） | `vertical_delay`（默认：1） | TSV相当于直通导线/隧道——信号以极小延迟通过 |
| **水平**（同层内路由） | `horizontal_delay`（默认：1000） | 层内金属互连布线，包括信号处理和路由决策——比TSV慢数个数量级 |

1000:1的比率反映了物理现实：TSV本质上是一段短垂直导体，而水平路由涉及金属走线、缓冲器和交换逻辑。

### 3.3 TSV共享

相对于整体传输时间，TSV传输信号极快。多个信号可以同时共享同一个TSV，不产生副作用。在本模型中，TSV竞争**不是**瓶颈。

> **实现说明**：代码中通过 `available_at` 时间戳追踪占用状态，但物理上假设TSV共享无冲突。

## 4. 失效模型

### 4.1 两种失效概念

系统区分两类失效：

1. **硬件失效** — TSV物理损坏或退化，永久不可用。由 `failure_rate` 参数建模，应用于单个TSV。

2. **信号传输失败** — 请求无法完成到顶层的传输（例如由于过多硬件失效导致无可行路径）。请求被标记为FAILED。

### 4.2 硬件失效模式

三种失效模式控制TSV何时发生物理失效：

| 模式 | 名称 | 描述 |
|------|------|------|
| **a** | 仅初始失效 | TSV仅在仿真启动时失效（制造缺陷） |
| **b** | 仅运行时失效 | TSV在仿真周期中动态失效（磨损、退化） |
| **c** | 混合失效 | 同时包含初始和运行时失效 |

**失效分布**：每个TSV以概率 `failure_rate` 独立失效（伯努利分布）。在运行时模式（b、c）中，每个仿真周期对每个未失效TSV执行此检查。

### 4.3 失效处理——冗余绕行

当路由算法遇到失效TSV时：

1. 搜索**同层**上最近的可用冗余TSV（按曼哈顿距离）
2. 计算绕行路径：当前位置 → 冗余TSV → 下一目标跳
3. 如果该层无可用冗余TSV，则路由必须寻找替代路径，否则请求失败

## 5. 请求模型

### 5.1 请求生成

每个仿真周期随机生成请求：

- **生成概率**：每周期约1%（平均每100个周期生成1个请求）
- **源点**：任意层上的随机位置，按热点分布加权
- **目标点**：**顶层**（z = L-1）上的随机位置，按热点分布加权

### 5.2 热点模型（负载局部性）

网格被划分为 (N/4) × (N/4) 个区域。每个区域被分配一个**热点权重** ∈ [0.5, 1.5]（均匀随机），代表负载局部性——芯片某些区域（如活跃核心、内存控制器）产生的流量多于其他区域。

源点和目标点通过**加权随机选择**区域，然后在所选区域内均匀随机选择位置来确定。

### 5.3 请求生命周期

每个请求经历以下状态：

```
PENDING → TRANSMITTING → COMPLETED
                      ↘ FAILED
```

1. **PENDING**：请求已创建，等待路由
2. **TRANSMITTING**：A*已计算路径，信号逐跳传输中
3. **COMPLETED**：信号到达顶层目标位置
4. **FAILED**：未找到可行路径，或传输中TSV失效后重路由失败

### 5.4 动态重路由

如果请求在传输过程中（TRANSMITTING状态）遭遇TSV失效，系统会尝试从**当前位置重新路由**到目标。这种动态重路由是核心功能——模拟了芯片从运行时失效中自适应恢复的能力。

如果重路由失败（无可行路径），请求标记为FAILED。

## 6. 路由算法

系统使用 **A*寻路算法** 计算三维网格中的路由：

- **启发函数**：曼哈顿距离 |Δx| + |Δy| + |Δz|（可采纳、一致）
- **代价函数**：动态代价，支持拥塞感知和可靠性感知（见6.1）
- **约束条件**：避开失效TSV；需要时使用冗余TSV绕行；强制路径长度和可靠性约束（见6.2）
- **可扩展性**：启发函数可通过 `set_heuristic()` 插拔替换

### 6.1 动态代价函数

在时间t穿过一个节点的代价为：

```
c(node, t) = c0(node) + λ1 · ccong(node, t) + λ2 · φ(pfail)
```

其中：
- **c0(node)**：基础延迟代价（根据移动方向为 `vertical_delay` 或 `horizontal_delay`）
- **ccong(node, t)**：拥塞代价——当前通过该节点的活跃传输路径数量
- **φ(pfail) = −ln(1 − pfail)**：风险惩罚——将节点失效概率映射为可加代价。这将乘积形式的可靠性转换为可加的求和形式
- **λ1**：拥塞权重（0 = 禁用）
- **λ2**：可靠性风险权重（0 = 禁用）

当 λ1 = 0 且 λ2 = 0（默认值）时，代价退化为原始的纯延迟模型。

### 6.2 路径约束

A*扩展过程中应用两个剪枝约束：

**最大路径长度（L_max）**：超过 `max_path_length` 跳数的路径被剪枝。设为0（默认）时不限制。

**最小可靠性（R_min）**：路径可靠性定义为：

```
R(P) = ∏(1 − pfail) ≥ R_min
```

取负对数后变为可加约束：

```
∑ φ(pfail) ≤ −ln(R_min)
```

累计风险超过此阈值的路径被剪枝。当 R_min = 0.0（默认）时不施加可靠性约束。

### 路径代价计算

对于路径 [p₀, p₁, ..., pₖ]：

```
总代价 = Σᵢ [c0(pᵢ, pᵢ₊₁) + λ1·ccong(pᵢ₊₁, t) + λ2·φ(pfail)]
```

其中 `c0(a, b)` = 同(x,y)时为 `vertical_delay`，否则为 `horizontal_delay`。

已完成请求的**延迟** = `完成时间 - 生成时间`。

## 7. 仿真循环

仿真运行可配置的周期数（默认：10,000,000）。每个周期按顺序执行四个步骤：

```
对每个周期：
    1. 生成新请求        （随机，约1%概率）
    2. 更新传输中请求    （推进在途信号一跳）
    3. 应用运行时失效    （仅模式b、c）
    4. 处理待处理请求    （通过A*计算路由）
```

## 8. 指标与输出

仿真收集并导出：

**统计摘要**（`{prefix}_summary.csv`）：
- 总请求数 / 完成数 / 失败数
- 成功率
- 延迟：最小值、最大值、平均值
- TSV失效数

**逐请求详情**（`{prefix}_requests.csv`）：
- 源点和目标点坐标
- 生成时间、完成时间
- 延迟、最终状态

## 9. 配置参数

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `num_layers` | 芯片层数 (L) | 2 |
| `grid_factor` | 网格尺寸因子 (N = 4 × grid_factor) | 4 |
| `failure_mode` | 硬件失效模式：a / b / c | a |
| `failure_rate` | 单TSV失效概率 | 1e-6 |
| `vertical_delay` | 垂直（层间）移动延迟 | 1 |
| `horizontal_delay` | 水平（层内）移动延迟 | 1000 |
| `simulation_cycles` | 总仿真周期数 | 100,000 |
| `output_prefix` | 输出CSV文件名前缀 | tsvra_output |
| `random_seed` | 随机数种子（0 = 随机设备） | 0 |
| `lambda1` | 拥塞权重（λ1） | 0.0（禁用） |
| `lambda2` | 可靠性风险权重（λ2） | 0.0（禁用） |
| `max_path_length` | 最大路径跳数（L_max，0=不限制） | 0 |
| `reliability_min` | 最小路径可靠性（R_min，0=不约束） | 0.0 |
